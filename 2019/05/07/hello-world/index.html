说说踩的坑

hexo命令

假设此时你已经建好了 Git 仓库，装好了 Hexo 并修改了其 _config.yml 文件。

  1. 首先初始化 Hexo，此时最好指定一个文件夹 folder，如果没有设置 folder，Hexo 默认在目前的文件夹建立网站。并转到该目录（cd floder）。
  2. 然后新建一篇文章，假设文章名为”new_article”。此时若启动服务器（hexo s），本地（localhost:4000）页面会出现这篇文章（/source/_post 目录下新建了 new_article.md 文件），但是还没有生成该文章的静态文件（可理解为还没有生成该文章对应的HTML文件）。建议在编写过程中打开本地服务，边写边看效果。
  3. 通过 hexo g 命令生成静态文件。此时 public 文件夹下以年月日命名的文件夹下便会生成对应文章（new_article.md）的HTML文件（index.html）。
  4. 通过 hexo d 命令将项目同步到 GitHub 。然后就可以在对应的网址上看见完整的博客内容了。

More info: Hexo 文档

_config.yml具体配置信息参考Hexo 系列之 _config.yml站点配置文件。

几点注意

  • 用markdawn语法写 \source\_posts 文件夹下的 md 文件，然后hexo g 、d，不要直接修改 \public\2019\05\07 下的 index.html 文件，index.html文件会随md文件的改变而改变。
  • 不要试图去手工改动 public 文件夹中的内容，public 文件夹是 hexo g 产生的。
