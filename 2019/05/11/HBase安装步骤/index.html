HBase安装步骤

发表于 2019-05-11 | 更新于: 2019-05-11
字数统计: 852 | 阅读时长 ≈ 3

一、安装HBase之前先安装Hadoop

1.下载Hadoop压缩包

2.cd /usr/storm

    使用tar zxvf hadoop-3.1.1.tar.gz解压

3.创建hadoop用户

    （1）useradd -m hadoop -s /bin/bash # 创建新用户hadoop
    （2）passwd hadoop
    （3） visudo
      root ALL=(ALL) ALL  然后在这行下面增加一行内容：hadoop ALL=(ALL) ALL （当中的间隔为tab）
      添加好内容后，先按一下键盘上的 ESC 键，然后输入 :wq (输入冒号还有wq，这是vi/vim编辑器的保存方法)，再按回车键保存退出就可以了。
      使用su hadoop即可登录hadoop用户名

4.$ sudo mv /usr/storm/hadoop-3.1.1/ /usr/storm/hadoop # 将文件夹名改为hadoop

    $ sudo chown -R hadoop:hadoop /usr/storm/hadoop # 修改文件权限
    如果不建立hadoop用户则   sudo chown -R root:hadoop /usr/storm/hadoop

5.环境变量

    修改 /etc/profile
    

    然后 source /etc/profil

6.测试是否安装成功

    root@storm-2:/usr/storm/hadoop/bin# hadoop version

7.配置集群/分布式环境

    （1）参考https://www.cnblogs.com/xzjf/p/7231519.html  
        修改/usr/storm/hadoop/etc/hadoop   中的
        core-site.xml、hdfs-site.xml、mapred-site.xml和yarn-site.xml
    （2）建立对应的文件夹   /usr/storm/hadoop/tmp/dfs/name        mkdir tmp等等
    （3）$ hdfs namenode -format # 首次运行需要执行初始化，之后不需要

8.进入/usr/storm/hadoop/sbin ./start-all.sh

     提示缺少变量命名

9.命令行直接写入变量

10.再次./start-all.sh提示没有配置JAVA_HOME

        进入/usr/storm/hadoop/etc/hadoop目录修改hadoop-env.sh
        vim ../etc/hadoop/hadoop-env.sh
        写入javahome

        nano ../etc/hadoop/hadoop-env.sh  也可以写入javahome，ssh不会提示要激活

11.进入/usr/storm/hadoop/sbin文件夹 输入 ./start-all.sh

        成功启动单机模式

12.配置Hadoop集群（和第7步结合）

    （1）按照上述做法（第七步配置四个xml文件，每个文件都不同），
            使得每个节点都是一个主节点（这种做法是错误的）
    （2）五台节点的    /usr/storm/hadoop/etc/hadoop   中的
            core-site.xml、hdfs-site.xml、mapred-site.xml和yarn-site.xml
            文件都应该是相同的（先配置好一台，将整个Hadoop文件夹复制到其他四台上即可）、
    （3）配置好这四个文件（一个节点上配置好直接复制到其他节点），再在workers文件写入
            storm-2/3/4/5（每个节点都要这样配置）
    （4）最后启动Hadoop集群，在主节点上启动，jps查看从节点，出现datanode证明集群配置成功
     注意：配置文件中最好不要用localhost，直接用节点名字

   注：创建Hadoop用户应该不需要

二、安装HBASE

  同上，先下载安装包并解压
  hbase-2.1.0文件夹改名为hbase（不是必需步骤）

1.在/etc/profile中配置好hbase的环境变量，然后source激活

2.配置/usr/storm/hbase/conf hbase-site.xml文件

3.配置/usr/storm/hbase/conf hbase-env.sh文件，导入：

    #Java环境
    export JAVA_HOME=/usr/storm/jdk1.8.0
    #使用HBASE自带的zookeeper管理集群
    export HBASE_MANAGES_ZK=true

4.配置/usr/storm/hbase/conf regionservers文件

5.将主节点配置好的hbase文件夹整个复制到其他节点上

    scp -r /usr/storm/hbase storm-2:/usr/storm/

6.启动： 进入hbase的bin文件夹下 ./start-hbase.sh

  注意：hbase-site.xml里面的文字格式（字体大小等）不要改变，否则
